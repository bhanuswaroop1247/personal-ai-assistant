# BASB – Building a Second Brain: Framework Summary

**Resource:** Productivity Systems  
**Type:** Framework summary – Tiago Forte's BASB  
**Last Updated:** 2025-01

---

## Core Thesis

Your brain is not optimized for storage — it's optimized for association and synthesis. Offloading captured information to an organized external system (your "Second Brain") frees mental RAM for creative thinking, pattern recognition, and producing output.

> "We have access to an unprecedented amount of information, yet feel overwhelmed rather than empowered." — Forte

---

## The CODE Framework

### C – Capture
Collect information that resonates — not everything, just what surprises you, challenges an assumption, or feels useful for future work.

**Rules:**
- Capture only what you actually resonate with (not "might be useful someday")
- Use a frictionless tool (phone note app, quick voice memo)
- Don't organize during capture — just collect

**My capture tools:** Phone notes → weekly transfer to PARA

### O – Organize
Move captured material into PARA:
- **Projects** — work with a deadline
- **Areas** — ongoing responsibilities
- **Resources** — reference by topic
- **Archive** — anything inactive

**Key principle:** Organize by *where you'll use it*, not by *what it is*. A note on Kalman filters goes in the Pupil Tracking project if actively using it, Resources > ML if it's general reference.

### D – Distill
Progressively summarize notes to extract the essential idea:
1. Raw note (captured)
2. Bold key passages (on first re-read)
3. Highlight bolded passages (on second re-read)
4. Executive summary in your own words

Don't distill everything — only notes you return to.

### E – Express
Use your Second Brain to produce outputs:
- Papers, code, talks, posts
- The system exists to enable creation, not just storage

---

## PARA in Practice

| Category | Time-bound? | Examples |
|----------|-------------|---------|
| Projects | Yes | Paper submission, pupil tracking app, interview prep |
| Areas | No | Research, career, health |
| Resources | No | ML notes, optics, writing tips |
| Archive | N/A | Old coursework, completed projects |

**Move freely between categories.** A Resource note can be promoted to a Project when you start using it actively. A Project becomes Archive when complete.

## The Intermediate Packets Concept

Rather than writing a paper from scratch, assemble it from "intermediate packets":
- Literature review bullets → Literature Review section
- OOB error observations → Methods/Results section
- Feature importance notes → Discussion

This is why detailed project notes pay dividends: you're not just documenting, you're pre-building the final output.

---

## Progressive Summarization vs. Highlighting

Naive knowledge management: highlight and never look again.  
Progressive summarization: each re-read passes the note through a filter, leaving only the essence. By the 4th level, a 2-page article is a 2-paragraph distillation in your own words — something you can use immediately in writing.

## What I've Adapted

From Forte's system:
- Use PARA structure (implemented here)
- Capture everything that surprises me in a single inbox
- Weekly review to process and route

My modifications:
- Technical notes get code snippets integrated (Forte's system is less technical)
- Research notes cross-reference with Zotero (his system doesn't address citation management)
- Markdown over proprietary note apps (portability > features)
